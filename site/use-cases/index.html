Use Cases

固定bufferから安全に読む・書く

offset更新、size不足、endianness、non-owning viewを明示して、バイナリ処理の事故を減らす。

CLI引数や設定文字列を読む

argc/argv、数値parse、port、IPv4、version、enum変換を段階的に分けて扱う。

失敗時に読みやすい診断を出す

byte差分、hex表現、契約違反、状態遷移の確認をテストと診断に寄せる。

小さい電文payloadを組み立てる

業務固有protocolをframework化せず、byte列構築、BCD、endian、overflow確認を組み合わせる。

後始末、期限、契約を局所化する

scope exit、move後reset、deadline、preconditionを小さく明示する。

template、tuple、variantの儀式を短くする

標準の強い型機能を隠さず、繰り返し現れる小さい型操作だけを名前にする。

文字列の小処理を読みやすくする

単純連結、ASCII限定処理、UTF-8検査、文字列から値への変換を混同しない。

値の範囲と表記を小さいAPIに固定する

裸のintや文字列で扱うとぶれやすいdomain値を、moduleごとの値型やparseで固定する。
