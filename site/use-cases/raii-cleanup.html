後始末、期限、契約を局所化する

scope exit、move後reset、deadline、preconditionを小さく明示する。

Decision Flow

  • scopeを抜ける時の後始末は scope を使う。
  • copy/move方針を型定義へ出すなら object を使う。
  • threadのjoin忘れを避けるなら concurrency を使う。
  • non-null pointer前提を局所化するなら pointer を使う。
  • 遅延生成した値の破棄境界を持つなら cache を使う。
  • 時間切れ判定は deadline を使う。
  • 常時評価したいpreconditionは contract を使う。
  • bufferやobject bytesの扱いは memory の境界を確認する。

Recipe

  • 所有・後始末の責務をscope単位で閉じる。
  • 型のcopy/move制約はclass宣言近くへ置く。
  • debug build限定では困る契約は contract へ寄せる。
