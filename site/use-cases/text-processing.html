文字列の小処理を読みやすくする

単純連結、ASCII限定処理、UTF-8検査、文字列から値への変換を混同しない。

Decision Flow

  • 文字列片を連結するだけなら string を使う。
  • ASCII前提のtrim、split、case変換なら ascii を使う。
  • UTF-8 byte列の妥当性を見るなら utf8 を使う。
  • 値への変換は parse やdomain moduleへ渡す。

Modules

ModuleRole
string複数文字列片の連結と追記。
asciiASCII前提のtrim、split、case、replace。
utf8UTF-8妥当性とcode point数。
parse文字列から数値・boolへの変換。

Recipe

  • 入力encodingの前提を先に決める。
  • ASCII限定の処理をUTF-8の一般処理として扱わない。
  • 連結とformatを混同せず、単純な部品結合だけ string に寄せる。
