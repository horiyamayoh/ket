CLI引数や設定文字列を読む

argc/argv、数値parse、port、IPv4、version、enum変換を段階的に分けて扱う。

Decision Flow

  • argvのoption有無と値取得は cli に任せる。
  • 数値やboolの文字列変換は parse を使う。
  • domain値は port、ipv4、version の専用moduleへ渡す。
  • 選択肢は enums のtable-based変換で固定する。
  • 任意値の後段処理は optional で合成する。

Modules

ModuleRole
cliargc/argvから小さいoptionを取り出す。
parsefrom_chars境界条件を固定した文字列変換。
portTCP/UDP port番号の範囲と表記を固定。
ipv4IPv4 dotted decimalとBE 32bit表現の相互変換。
versionnumeric version tripletのparse/format/compare。
enumsenum classと文字列・flagsの変換。

Recipe

  • cliで文字列を取り出す。
  • 値の種類ごとにparse先moduleを変える。
  • 失敗時はどの段階で失敗したかを呼び出し側で表示する。
