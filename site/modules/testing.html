Use This When

  • GoogleTestでbyte列やhex期待値を読みやすく比較したい時。
  • 失敗時にoffset差分を見たい時。

Do Not Use This When

  • library本体コードから使う時。
  • GoogleTest以外のtest frameworkを使う時。

API Details

BytesEqual function

Signature
::testing::AssertionResult BytesEqual(const std::uint8_t* expected, std::size_t expected_size, const std::uint8_t* actual, std::size_t actual_size)
Brief2つのbyte列をGoogleTest assertionとして比較。
Parameters
  • in expected: 期待するbyte列。`expected_size == 0` の場合はnullptrを空列として許容。
  • in expected_size: `expected`のバイト数。
  • in actual: 実際のbyte列。`actual_size == 0` の場合はnullptrを空列として許容。
  • in actual_size: `actual`のバイト数。
Return / retval
  • success: byte列の長さと内容が一致。
  • failure: nullptr+非0サイズ、長さ差、または内容差。長さ差と内容差のfailure messageに差分offsetとexpected/actual hexを含む。nullptr+非0サイズのfailure messageには不正な引数名とsizeを含む。
Pre`expected`と`actual`は、それぞれのsizeが0でない場合にsizeバイト以上読み取り可能な 配列を指す。nullptr+非0サイズはfailureとして扱う。
Post引数と外部状態の変更なし。成功時は診断用stringを生成しない。
Example
const std::uint8_t expected[] = {0x12U, 0x34U};
const std::uint8_t actual[] = {0x12U, 0x34U};
EXPECT_TRUE(ket::testing::BytesEqual(expected, 2U, actual, 2U));

HexEqual function

Signature
::testing::AssertionResult HexEqual(std::string_view expected_hex, const std::uint8_t* actual, std::size_t actual_size)
Briefhex文字列で表した期待byte列と実際のbyte列をGoogleTest assertionとして比較。
Parameters
  • in expected_hex: 期待するbyte列のhex文字列。大文字小文字を許容し、ASCII whitespaceは読み飛ばす。
  • in actual: 実際のbyte列。`actual_size == 0` の場合はnullptrを空列として許容。
  • in actual_size: `actual`のバイト数。
Return / retval
  • success: `expected_hex`が妥当なhexで、復号後の長さと内容が`actual`と一致。
  • failure: 不正hex、nullptr+非0サイズ、長さ差、または内容差。failure messageに 不正hexの位置、nullptr引数名、または差分offsetとexpected/actual hexを含む。
Pre`expected_hex`は呼び出し中有効な文字列範囲。`actual`は`actual_size`が0でない場合に `actual_size`バイト以上読み取り可能な配列を指す。nullptr+非0サイズはfailureとして扱う。
Post引数と外部状態の変更なし。成功時は診断用stringを生成しない。
Example
const std::uint8_t actual[] = {0x12U, 0x34U};
EXPECT_TRUE(ket::testing::HexEqual("12 34", actual, 2U));
