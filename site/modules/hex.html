ket::hex

byte列、整数、診断用hexdumpをhex文字列へ変換する。

Use This When

  • byte列の期待値や差分を人間が読みたい時。
  • diagnostic用hexdumpを小さく出したい時。

Do Not Use This When

  • binary codec全般やbase64等を扱う時。
  • 高性能dump viewerを作りたい時。

Format function

std::string Format(std::uint64_t value, unsigned min_width = 0, LetterCase letter_case = LetterCase::kUpper)

整数のhex文字列化。

Parameters

  • in value: 文字列化対象の整数。
  • in min_width: 出力する最小hex桁数。桁数不足分は左側を0で埋める。
  • in letter_case: 10以上のhex digitに使う英字の大文字小文字。

Return / retval

  • value: hex文字列。`value`の実桁が`min_width`を超える場合は実桁を保持。

Contract

prepost
なし。引数と外部状態の変更なし。

Notes

  • `value == 0` では `min_width == 0` でも1桁の`"0"`を返す。
  • std::stringの確保があるためnoexceptなし。

Example

const auto text = ket::hex::Format(0x2aU, 4U, ket::hex::LetterCase::kLower);
// text == "002a"

Decode function

std::optional<std::vector<std::uint8_t>> Decode(std::string_view text)

hex文字列のbyte列変換。

Parameters

  • in text: 変換対象のhex文字列。

Return / retval

  • value: 変換後のbyte列。空入力またはASCII whitespaceのみの入力では空vector。
  • std::nullopt: 奇数桁、不正hex文字、またはASCII whitespace以外の非hex文字。

Contract

prepost
なし。空入力と不正入力は戻り値で区別。引数と外部状態の変更なし。

Notes

  • ASCII whitespaceは`0x09`から`0x0d`と`0x20`のみ許容。
  • separatorつきhexの自動parseは行わない。separatorは不正文字として扱う。
  • std::vectorの確保があるためnoexceptなし。

Example

const auto bytes = ket::hex::Decode("de ad BE EF");
// bytes == std::optional<std::vector<std::uint8_t>>({0xde, 0xad, 0xbe, 0xef})

Dump function

std::string Dump(const std::uint8_t* data, std::size_t size)

byte列の固定形式hexdump生成。

Parameters

  • in data: dump対象のbyte列。
  • in size: `data`のバイト数。

Return / retval

  • value: 固定形式hexdump文字列。`size == 0` の場合は空文字列。

Contract

prepost
`data`は`size`バイト以上読み取り可能な配列を指す。`nullptr` は `size == 0` の場合だけ許容し、`nullptr` かつ非0 sizeはprecondition違反。引数と外部状態の変更なし。

Notes

  • 16 bytes/row、最小8桁zero-pad lower hex offset、lower hex byte、8 byteごとの 追加space、`|...|`で囲むASCII previewの固定形式。ASCII previewは`0x20`から`0x7e` をそのまま表示し、それ以外は`.`。最終行のhex欠落分はspace padding。行末のnewlineは付けない。
  • std::stringの確保があるためnoexceptなし。

Example

const std::uint8_t data[] = {0x00U, 0x41U};
const auto text = ket::hex::Dump(data, 2U);
// text == "00000000  00 41                                             |.A|"

DumpMemory function

std::string DumpMemory(const void* data, std::size_t size)

型なしメモリ範囲の固定形式hexdump生成。

Parameters

  • in data: dump対象のメモリ範囲。
  • in size: `data`のバイト数。

Return / retval

  • value: 固定形式hexdump文字列。`size == 0` の場合は空文字列。

Contract

prepost
`data`は`size`バイト以上読み取り可能なメモリ範囲を指す。`nullptr` は `size == 0` の場合だけ許容し、`nullptr` かつ非0 sizeはprecondition違反。引数と外部状態の変更なし。

Notes

  • C API境界の`void*` bufferをbyte列としてdumpするための明示API。
  • std::stringの確保があるためnoexceptなし。

Example

const std::uint32_t value = 0x12345678U;
const auto text = ket::hex::DumpMemory(&value, sizeof(value));
// textは`value`のobject representationをdumpした文字列
