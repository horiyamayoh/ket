ket::numeric

整数overflow、alignment、safe castの境界確認を扱う。

Module Facts

Headermodules/numeric/ket_numeric.h
Statusverified
C++ MinC++11
TestsGoogleTest + C++11 compile-only
Progress noteoverflow、align、cast の境界確認
Drop-in`ket_numeric.h` を対象プロジェクトへコピー。ヘッダオンリーmodule。
Dependencies標準ライブラリのみ。 他のket moduleへの依存なし。
Namespace公開API：ket::numeric 内部実装：ket::numeric::detail

Use This When

  • signed/unsignedや幅変換の境界を毎回書きたくない時。
  • overflowやalignment 0を失敗として扱いたい時。

Do Not Use This When

  • 任意精度整数やdecimal演算が必要な時。
  • 標準の単純なcastで意図が十分明確な時。

Views

バイナリ・byte列 値型・validation小さい電文payloadを組み立てる 値の範囲と表記を小さいAPIに固定する

InRange function

template <typename To, typename From> constexpr bool InRange(From value) noexcept

値が指定変換先整数型の範囲内かの判定。

Parameters

  • in value: 判定対象値。

Return / retval

  • true: `value`が`To`の表現範囲内。
  • false: `value`が`To`の表現範囲外。

Contract

prepost
`To`と`From`はboolとtext character型を除くintegral型。対象外型はcompile error。引数と外部状態の変更なし。

Notes

  • signed/unsignedの組み合わせでも比較前に十分広い整数表現へ変換。

Example

const auto ok = ket::numeric::InRange<int>(42U);
const auto ng = ket::numeric::InRange<unsigned>(-1);
// ok == true, ng == false
