numeric version tripletのparse、format、compareを扱う。

Use This When

  • major.minor.patchの数値tripletだけを扱いたい時。
  • leading zeroやoverflowの扱いを固定したい時。

Do Not Use This When

  • full SemVer、pre-release、build metadataが必要な時。
  • 任意形式のversion rangeを扱いたい時。

API Details

Triplet type

struct Triplet

`major.minor.patch`の数値3要素。

Notes

  • 各要素は`std::uint32_t`として保持。SemVerのprereleaseやbuild metadataは保持なし。

Parse function

std::optional<Triplet> Parse(std::string_view text) noexcept

`major.minor.patch`形式のnumeric version triplet parse。

Parameters

  • in text: parse対象の文字列。

Return / retval

  • value: parse済みの数値3要素。
  • std::nullopt: 空、要素不足/過多、空要素、非数字、符号、空白、overflow、 複数桁のleading zero、またはprerelease/build metadata。

Contract

prepost
なし。不正入力は失敗値として扱う。引数と外部状態の変更なし。

Notes

  • 各要素は`std::uint32_t`範囲の10進ASCII数字列。`0`単体は許可。

Example

const auto value = ket::version::Parse("1.20.300");
// value.has_value() == true
// value->major == 1U
// value->minor == 20U
// value->patch == 300U

const auto invalid = ket::version::Parse("1.2.3-alpha");
// invalid == std::nullopt

numeric version tripletの文字列化。

Parameters

prepost

Notes

Example

// text == "1.20.300"

Compare function

int Compare(Triplet a, Triplet b) noexcept

numeric version tripletの要素順比較。

Parameters

  • in a: 比較左辺の数値3要素。
  • in b: 比較右辺の数値3要素。

Return / retval

  • negative: `a`が`b`より小さい。
  • 0: `a`と`b`が等しい。
  • positive: `a`が`b`より大きい。

Contract

prepost
なし。引数と外部状態の変更なし。

Notes

  • 比較順はmajor、minor、patch。

Example

const auto order = ket::version::Compare({1U, 2U, 3U}, {1U, 3U, 0U});
// order < 0
