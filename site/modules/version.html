numeric version tripletのparse、format、compareを扱う。

Use This When

  • major.minor.patchの数値tripletだけを扱いたい時。
  • leading zeroやoverflowの扱いを固定したい時。

Do Not Use This When

  • full SemVer、pre-release、build metadataが必要な時。
  • 任意形式のversion rangeを扱いたい時。

Parse function

Signature
std::optional<Triplet> Parse(std::string_view text) noexcept
Brief`major.minor.patch`形式のnumeric version triplet parse。
Parameters
  • in text: parse対象の文字列。
Return / retval
  • value: parse済みの数値3要素。
  • std::nullopt: 空、要素不足/過多、空要素、非数字、符号、空白、overflow、 複数桁のleading zero、またはprerelease/build metadata。
Preなし。不正入力は失敗値として扱う。
Post引数と外部状態の変更なし。
Notes
  • 各要素は`std::uint32_t`範囲の10進ASCII数字列。`0`単体は許可。
Example
const auto value = ket::version::Parse("1.20.300");
// value.has_value() == true
// value->major == 1U
// value->minor == 20U
// value->patch == 300U

const auto invalid = ket::version::Parse("1.2.3-alpha");
// invalid == std::nullopt
