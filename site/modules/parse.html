ket::parse

from_chars境界条件とbool parseを固定する。

Categories文字列・text・parse 入力境界・外部境界
Use casesCLI引数や設定文字列を読む 文字列の小処理を読みやすくする 値の範囲と表記を小さいAPIに固定する

Use This When

  • 数値やboolのparse規則を呼び出し側ごとにぶらしたくない時。
  • 空白、符号、overflowなどの扱いを固定したい時。

Do Not Use This When

  • domain値の専用parseがある時。
  • locale依存parseやformatが必要な時。

TryInt function

Signature
template <typename T> bool TryInt(std::string_view text, T& out) noexcept
Brief10進文字列の符号付き整数変換。
Parameters
  • in text: 変換対象の文字列。
  • out out: 変換後の値。成功時だけ更新。
Return / retval
  • true: `text`を完全消費し、`T`の範囲内で変換成功。
  • false: 空文字列、空白、部分消費、不正文字、先頭の`+`、または範囲外。
Preなし。失敗条件は戻り値で扱う。
Post成功時は`out`を変換結果へ更新。失敗時は`out`を入力時の値に保持。
Example
int value = 0;
const auto ok = ket::parse::TryInt("-42", value);
// ok == true, value == -42

Brief16進文字列の整数変換。

  • out out: 変換後の値。成功時だけ更新。
Return / retval
  • true: prefix除去後の文字列を完全消費し、`T`の範囲内で変換成功。
  • false: 空文字列、prefixのみ、空白、符号文字、部分消費、不正hex、または範囲外。

Post成功時は`out`を変換結果へ更新。失敗時は`out`を入力時の値に保持。
Notes
  • 16進表記は符号文字を扱わない。signed `T`でも非負のhex桁列だけを変換対象とする。
Example
std::uint32_t value = 0;
const auto ok = ket::parse::TryHex("0xff", value);
// ok == true, value == 255
