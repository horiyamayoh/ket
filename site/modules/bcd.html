ket::bcd

packed BCDと10進整数・10進文字列の相互変換を扱う。

Use This When

  • 電文や古い外部仕様でpacked BCD fieldが必要な時。
  • nibble妥当性や先頭ゼロ保持を呼び出し側に散らしたくない時。

Do Not Use This When

  • 通常の10進文字列formatだけで十分な時。
  • 任意精度decimal演算や金額計算をしたい時。

FromInt function

template <typename Packed> constexpr std::optional<Packed> FromInt(int value) noexcept

固定幅パックBCDへの10進整数変換。

Parameters

  • in value: 変換対象の10進整数。

Return / retval

  • value: 変換後のパックBCD値。
  • std::nullopt: 負数、または`Packed`の固定幅桁数を超える値。

Contract

prepost
なし。負数と桁数超過は失敗値として扱う。引数と外部状態の変更なし。

Notes

  • `std::uint8_t`は2桁、`std::uint16_t`は4桁、`std::uint32_t`は8桁の fixed-width packed BCDとして表現。

Example

const auto value = ket::bcd::FromInt<std::uint8_t>(42);
// value == std::optional<std::uint8_t>(0x42)

Format function

std::optional<std::string> Format(const std::uint8_t* data, std::size_t size)

任意バイト長パックBCDの10進文字列変換。

Parameters

  • in data: 変換対象のパックBCD列。
  • in size: `data`のバイト数。

Return / retval

  • value: 変換後の10進文字列。
  • std::nullopt: `nullptr`、空入力、または不正nibble。

Contract

prepost
`data`は`size`バイト以上読み取り可能な配列を指す。`nullptr` と空入力は失敗値として扱う。引数と外部状態の変更なし。

Notes

  • 入力BCDの桁数を保ち、先頭の0も文字'0'として出力。
  • std::stringの確保があるためnoexceptなし。

Example

const std::uint8_t data[] = {0x00U, 0x42U};
const auto value = ket::bcd::Format(data, 2U);
// value == std::optional<std::string>("0042")

Parse function

std::optional<std::vector<std::uint8_t>> Parse(std::string_view text)

10進文字列の任意バイト長パックBCD変換。

Parameters

  • in text: 変換対象の10進文字列。

Return / retval

  • value: 変換後のパックBCD列。
  • std::nullopt: 空入力、または10進数字以外を含む入力。

Contract

prepost
なし。空入力と10進数字以外は失敗値として扱う。引数と外部状態の変更なし。

Notes

  • 偶数桁は2桁ずつpacked BCDへ変換し、奇数桁は先頭に0を補って変換。
  • std::vectorの確保があるためnoexceptなし。

Example

const auto value = ket::bcd::Parse("123");
// value == std::optional<std::vector<std::uint8_t>>({0x01, 0x23})
