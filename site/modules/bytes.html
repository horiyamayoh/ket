ket::bytes

owning byte payloadを短いAPIで構築する。

Module Facts

Headermodules/bytes/ket_bytes.h
Statusverified
C++ MinC++17
TestsGoogleTest
Progress note可変長byte payloadの構築
Drop-in`ket_bytes.h` を対象プロジェクトへコピー。ヘッダオンリーmodule。
Dependencies標準ライブラリのみ。 他のket moduleへの依存なし。
Namespace公開API：ket::bytes 内部実装：ket::bytes::detail
Categoriesバイナリ・byte列
Use cases小さい電文payloadを組み立てる 固定bufferから安全に読む・書く

Use This When

  • std::vector<std::uint8_t>へfieldを順に追加したい時。
  • BE/LEやbyte列追加の意図を呼び出し側で読みたい時。

Do Not Use This When

  • 固定bufferへ書く必要がある時。
  • 巨大なserialization frameworkが必要な時。

Append function

Signature
inline void Append(std::vector<std::uint8_t>& dst, const std::uint8_t* data, std::size_t size)
Brief任意byte列の末尾追記。
Parameters
  • in,out dst: 追記先のbyte列。
  • in data: 追記するbyte列の先頭。`size == 0`の場合だけnullptr可。
  • in size: 追記するbyte数。
Return / retval
  • void: 戻り値なし。
Pre`dst`は有効なstd::vectorオブジェクト。`data`は`size`バイト以上読み取り可能な 配列を指す。`data[0..size)`は`dst`の内部storageと重ならない。`Append(nullptr, 0)`はno-op、 `Append(nullptr, size > 0)`はprecondition違反。
Post`dst`の既存内容を保持し、`size`が0でなければ末尾に`data[0..size)`を同じ順序で追加。
Notes
  • self-appendは未対応。`data`が`dst`の要素を指す場合の動作は未定義。
  • raw byte列を扱うC API境界に近い入力のため、nullable条件付きのポインタを使う。
  • std::vectorの確保があるためnoexceptなし。
Example
const std::uint8_t data[] = {0x01U, 0x02U};
std::vector<std::uint8_t> bytes{0xF0U};
ket::bytes::Append(bytes, data, 2U);
// bytes == {0xF0, 0x01, 0x02}

AppendAscii function

Signature
Builder& AppendAscii(std::string_view text)
BriefASCII byte列として扱う文字列片の末尾追記。
Parameters
  • in text: 追記するASCII文字列片。
Return / retval
  • *this: 追記後のbuilder。
Pre`text`の各要素はASCII byte列。`text`の参照範囲は内部bufferと重ならない。 UTF-8 validationやencoding変換は行わない。
Post内部bufferの既存内容を保持し、末尾に`text`の各byteを同じ順序で追加。
Notes
  • ASCII性はprecondition。各byteを検査せず追記し、非ASCII byteも変換しない。
  • self-appendは未対応。`text`が内部buffer由来の場合の動作は未定義。
  • std::vectorの確保があるためnoexceptなし。
Example
ket::bytes::Builder builder;
builder.AppendAscii("AZ");
// builder.Buffer() == {0x41, 0x5A}
