Use This When

  • 小さいツールで --key value や --key=value を扱いたい時。
  • CLI frameworkを入れるほどではない時。

Do Not Use This When

  • subcommand、help生成、shell風quote処理が必要な時。
  • 設定fileや環境変数の統合管理をしたい時。

API Details

ArgvView type

class ArgvView

argc/argvのnon-owning view。

Notes

  • 各argumentはnull終端C文字列として扱う。`argv[i] == nullptr`は空argument。
  • `argv` pointer配列と各非null文字列は、viewと戻り値の利用中に有効なまま保持。

argv_ function

ArgvView(int argc, const char* const* argv) noexcept : size_(argc > 0 && argv != nullptr ? static_cast<std::size_t>(argc) : 0U), argv_(argc > 0 && argv != nullptr ? argv : nullptr)

argc/argv viewの構築。

Parameters

  • in argc: argv要素数。
  • in argv: argv配列先頭。`argc > 0 && argv == nullptr`は空view。

Contract

prepost
`argv`が非nullの場合、`argc`個のポインタを読み取り可能。各非null文字列はnull終端。`argc < 0`または`argv == nullptr`の場合は空view。 文字列領域とpointer配列の所有権は取得なし。

Example

const char* argv[] = {"tool", "--help"};
const auto args = ket::cli::ArgvView(2, argv);
// args.Size() == 2
// args.ProgramNameOrEmpty() == "tool"

HasOption function

[[nodiscard]] inline bool HasOption(ArgvView args, std::string_view name) noexcept

  • false: option不在、`name`不正、またはbare `--`より後ろにだけ存在。

// has_help == true

OptionValue function

[[nodiscard]] inline std::optional<std::string_view> OptionValue(ArgvView args, std::string_view name) noexcept

option値の取得。

Parameters

  • in args: 検索対象argv view。
  • in name: 検索するoption名。`"--"`で始まり、名前部分が非空で、`=`を含まない名前のみ有効。

Return / retval

  • value: `--key value`または`--key=value`の値。
  • std::nullopt: option不在、`name`不正、値なし、次要素が別option、またはbare `--`より後ろにだけ存在。

Contract

prepost
なし。program nameは検索対象外。引数と外部状態の変更なし。戻り値はargv文字列領域を参照。

Notes

  • 重複optionは先の出現を採用し、先の出現に値がない場合は後続の値を見ない。
  • inline値とseparate値は空文字列を有効な値として返す。shell quote展開は扱わない。

Example

const char* argv[] = {"tool", "--id", "123"};
const auto args = ket::cli::ArgvView(3, argv);
const auto id = ket::cli::OptionValue(args, "--id");
// id == std::optional<std::string_view>("123")

Example

const char* argv[] = {"tool", "--id"};
const auto args = ket::cli::ArgvView(2, argv);

PositionalArguments function

[[nodiscard]] inline std::vector<std::string_view> PositionalArguments(ArgvView args)

schemaなしのpositional argument取得。

Parameters

  • in args: 取得対象argv view。

Return / retval

  • value: program nameとoption形argumentを除くargumentの列。bare `--`以降はすべて保持。

Contract

prepost
なし。option schemaは持たず、separate option値はpositional候補として保持。引数と外部状態の変更なし。戻り値の各viewはargv文字列領域を参照。

Notes

  • 結果vectorの確保があるためnoexceptなし。`--id 123`の`123`はschemaなしでは値か positionalか判別しないため保持。

Example

const char* argv[] = {"tool", "input.txt", "--verbose", "output.txt"};
const auto args = ket::cli::ArgvView(4, argv);
const auto positional = ket::cli::PositionalArguments(args);
// positional == {"input.txt", "output.txt"}
