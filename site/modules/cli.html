ket::cli

argc/argvからoption有無、値、positional argumentsを取得する。

Use This When

  • 小さいツールで --key value や --key=value を扱いたい時。
  • CLI frameworkを入れるほどではない時。

Do Not Use This When

  • subcommand、help生成、shell風quote処理が必要な時。
  • 設定fileや環境変数の統合管理をしたい時。

Notes

argv_ function

Signature
ArgvView(int argc, const char* const* argv) noexcept : size_(argc > 0 && argv != nullptr ? static_cast<std::size_t>(argc) : 0U), argv_(argc > 0 && argv != nullptr ? argv : nullptr)
Briefargc/argv viewの構築。
Parameters
  • in argc: argv要素数。
  • in argv: argv配列先頭。`argc > 0 && argv == nullptr`は空view。
Pre`argv`が非nullの場合、`argc`個のポインタを読み取り可能。各非null文字列はnull終端。
Post`argc < 0`または`argv == nullptr`の場合は空view。 文字列領域とpointer配列の所有権は取得なし。
Example

const auto args = ket::cli::ArgvView(2, argv);
// args.Size() == 2
// args.ProgramNameOrEmpty() == "tool"

Return / retval

Notes
  • 重複optionは先の出現を採用し、先の出現に値がない場合は後続の値を見ない。
  • inline値とseparate値は空文字列を有効な値として返す。shell quote展開は扱わない。
Example

const auto args = ket::cli::ArgvView(3, argv);
const auto id = ket::cli::OptionValue(args, "--id");
// id == std::optional<std::string_view>("123")
