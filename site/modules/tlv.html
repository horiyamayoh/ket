ket::tlv

type:uint16、length:uint32、value bytesの1 recordをencode/decodeする。

Use This When

  • 単純なTLV recordのwire format境界を固定したい時。
  • decode失敗時の出力不変性を呼び出し側で利用したい時。

Do Not Use This When

  • schema languageや複数record iteratorが必要な時。
  • ASN.1やBER/DERのような汎用TLV仕様を扱う時。

Encode function

std::vector<std::uint8_t> Encode(std::uint16_t type, const std::uint8_t* value, std::size_t value_size)

1つのTLV recordをbyte列へencode。

Parameters

  • in type: wire headerへbig-endianで格納するtype。
  • in value: 追加するvalue bytes。`value_size == 0`の場合のみ`nullptr`可。
  • in value_size: 追加するvalue byte数。wire headerへbig-endian uint32で格納。

Return / retval

  • value: `type:uint16 big-endian` + `length:uint32 big-endian` + value bytes。

Contract

prepost
`0 < value_size && value_size <= UINT32_MAX`の場合、`value`は`value_size` バイト以上読み取り可能な配列を指す。`value == nullptr && value_size > 0`はprecondition違反。引数と外部状態の変更なし。

Notes

  • `value_size > UINT32_MAX`、またはrecord sizeがstd::vector::max_size()を超える場合は std::length_error送出。
  • std::vectorの確保があるためnoexceptなし。

Example

const std::uint8_t value[] = {0xAAU};
const auto record = ket::tlv::Encode(0x1234U, value, 1U);
// record.size() == 7, record[0] == 0x12, record[6] == 0xAA

Append function

void Append(std::vector<std::uint8_t>& dst, std::uint16_t type, const std::uint8_t* value, std::size_t value_size)

1つのTLV recordを既存byte列の末尾へ追加。

Parameters

  • in,out dst: 追加先のbyte列。
  • in type: wire headerへbig-endianで格納するtype。
  • in value: 追加するvalue bytes。`value_size == 0`の場合のみ`nullptr`可。
  • in value_size: 追加するvalue byte数。wire headerへbig-endian uint32で格納。

Return / retval

  • void: 戻り値なし。

Contract

prepost
`dst`は有効なstd::vector。`0 < value_size && value_size <= UINT32_MAX`の場合、 `value`は`value_size`バイト以上読み取り可能な配列を指す。`value`は`dst`の内部storageを指していてもよい。 `value == nullptr && value_size > 0`はprecondition違反。`dst`の既存内容を保持し、末尾にTLV recordを追加。

Notes

  • `value_size > UINT32_MAX`、record size、または出力合計がstd::vector::max_size()を超える場合は std::length_error送出。
  • std::vectorの確保があるためnoexceptなし。
  • 例外時は`dst`を変更しない強い例外保証。

Example

std::vector<std::uint8_t> output;
const std::uint8_t value[] = {0xAAU};
ket::tlv::Append(output, 0x1234U, value, 1U);
// output.size() == 7, output[0] == 0x12, output[6] == 0xAA

TryDecode function

bool TryDecode(const std::uint8_t* data, std::size_t size, DecodeResult& out) noexcept

入力先頭の1つのTLV recordをdecode。

Parameters

  • in data: decode対象のbyte列。`nullptr`の場合は`size`によらず失敗。
  • in size: `data`のbyte数。
  • out out: decode成功時に先頭recordのviewと消費byte数を格納。

Return / retval

  • true: 先頭1 recordをdecodeし、`out`を更新。
  • false: `data == nullptr`、6 bytes未満、declared length過大、 または消費byte数のstd::size_t overflow。`out`は不変。

Contract

prepost
`data != nullptr`の場合、`data`は`size`バイト以上読み取り可能な配列を指す。成功時だけ`out`を更新。失敗時は`out`を変更なし。

Notes

  • `out.view.value`は入力bufferを指すnon-owning pointer。 decode後の参照有効性は入力buffer lifetimeに依存。

Example

const std::uint8_t record[] = {0x12U, 0x34U, 0x00U, 0x00U, 0x00U, 0x01U, 0xAAU};
ket::tlv::DecodeResult result{};
const auto ok = ket::tlv::TryDecode(record, sizeof(record), result);
// ok == true, result.view.type == 0x1234, result.view.value_size == 1
