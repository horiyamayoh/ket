ket::uuid

canonical hyphen形式のUUID文字列と16 byte表現を相互変換する。

Use This When

  • UUID literalをcanonical hyphen形式として検証したい時。
  • 16 byte表現からlower-case固定の文字列を生成したい時。

Do Not Use This When

  • UUID生成、URN形式、version/variant validationが必要な時。
  • 任意のidentifier形式を汎用的に扱いたい時。

API Details

Uuid type

struct Uuid

UUID の16 byte表現。

Notes

  • `bytes[0]`はcanonical文字列の先頭2桁、`bytes[15]`は末尾2桁に対応。

Parse function

std::optional<Uuid> Parse(std::string_view text) noexcept

canonical hyphen形式 UUID 文字列のparse。

Parameters

  • in text: parse対象の文字列。

Return / retval

  • value: parse後の UUID。
  • std::nullopt: 長さ不一致、hyphen位置不一致、不正hex、brace/URN形式、 またはcanonical hyphen形式以外。

Contract

prepost
なし。canonical hyphen形式以外は失敗値として扱う。引数と外部状態の変更なし。version/variant bit の妥当性検証なし。

Notes

  • hex は upper/lower の両方を受け付ける。

Example

const auto value = ket::uuid::Parse("00112233-4455-6677-8899-aabbccddeeff");
// value.has_value() == true

Format function

std::string Format(const Uuid& value)

UUID のcanonical hyphen形式文字列化。

Parameters

  • in value: 文字列化対象の UUID。

Return / retval

  • value: lower-case 固定のcanonical hyphen形式文字列。

Contract

prepost
`value.bytes` は16 byteの UUID 表現。引数と外部状態の変更なし。

Notes

  • std::stringの確保があるためnoexceptなし。

Example

const auto value = ket::uuid::Uuid{{0x00U, 0x11U, 0x22U, 0x33U, 0x44U, 0x55U,
                                   0x66U, 0x77U, 0x88U, 0x99U, 0xaaU, 0xbbU,
                                   0xccU, 0xddU, 0xeeU, 0xffU}};
const auto text = ket::uuid::Format(value);
// text == "00112233-4455-6677-8899-aabbccddeeff"
