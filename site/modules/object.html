ket::object

copy/move制御とmove後resetを型定義の近くに集約する。

Use This When

  • copy禁止やmove onlyの意図を型で明示したい時。
  • move後に値をresetする小さいラッパが必要な時。

Do Not Use This When

  • =deleteを直接書く方が十分明確な時。
  • 所有権modelやresource frameworkを作りたい時。

NonCopyable type

class NonCopyable

派生型のcopyを禁止する継承用mixin。

Notes

  • moveは禁止しない。派生型のmove可否は派生型自身の特殊メンバに従う。
  • 空baseとして使うため、data memberと比較演算は持たない。

Contract

ResetOnMove type

template <typename T> class ResetOnMove

move 後のsourceを値初期化状態へ戻す小値 wrapper。

Notes

  • `T`はdefault constructible、move constructible、move assignableである必要あり。
  • 比較演算は宣言せず、比較方針は利用側の型へ委ねる。

ResetOnMove function

ResetOnMove() = default

値初期化した保持値で構築。

Contract

prepost
`T{}`が有効。`Get()`は値初期化された`T`への参照を返す。

Example

ket::object::ResetOnMove<int> value;
// value.Get() == 0

ResetOnMove function

explicit ResetOnMove(T value)

初期値を保持して構築。

Parameters

  • in value: 保持する初期値。

Contract

prepost
`T`は`value`からmove構築可能。`Get()`は渡された値からmove構築した`T`への参照を返す。

Example

noexcept function

ResetOnMove(ResetOnMove&& other) noexcept( std::is_nothrow_move_constructible<T>::value && std::is_nothrow_default_constructible<T>::value && std::is_nothrow_move_assignable<T>::value)

move 構築後にsourceを値初期化状態へ戻す。

Parameters

  • in,out other: move元wrapper。

Contract

prepost
`other`は有効なwrapper。`T{}`によるresetが有効。構築先はmove前の`other.Get()`の値を保持し、`other.Get()`は`T{}`で代入された状態。

Notes

  • 例外発生時の構築先とsourceの状態は`T`のmove/reset操作の保証に従う。

Example

ket::object::ResetOnMove<int> source(42);
ket::object::ResetOnMove<int> moved(std::move(source));
// moved.Get() == 42, source.Get() == 0
