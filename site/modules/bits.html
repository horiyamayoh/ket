ket::bits

bit、nibble、mask、rotateの境界処理を名前付きで扱う。

Module Facts

Headermodules/bits/ket_bits.h
Statusverified
C++ MinC++11
TestsGoogleTest + C++11 compile-only
Progress notebit/nibble/mask/rotateの境界処理
Drop-in`ket_bits.h` を対象プロジェクトへコピー。ヘッダオンリーmodule。
Dependencies標準ライブラリのみ。 他のket moduleへの依存なし。
Namespace公開API：ket::bits 内部実装：ket::bits::detail
Categoriesバイナリ・byte列
Use cases小さい電文payloadを組み立てる 値の範囲と表記を小さいAPIに固定する

Use This When

  • shift幅やmask境界を毎回手書きしたくない時。
  • nibble抽出やpackの意図を読みやすくしたい時。

Do Not Use This When

  • C++20の標準bit APIで同じ意図が十分明確な時。
  • CPU命令やSIMD最適化を直接制御したい時。

Notes

  • unsigned integralを主対象にし、未定義動作につながるshiftを避ける。

TryPackNibbles function

Signature
inline bool TryPackNibbles(std::uint8_t high, std::uint8_t low, std::uint8_t& out) noexcept
Brief2つのnibbleからbyte値を生成。
Parameters
  • in high: 上位nibbleとして格納する値。
  • in low: 下位nibbleとして格納する値。
  • out out: 成功時の生成byte値。失敗時は変更なし。
Return / retval
  • true: `high`と`low`がnibble範囲内で、`out`を更新。
  • false: `high`または`low`がnibble範囲外。`out`は入力時の値を保持。
Pre`out`は有効なstd::uint8_t参照。
Post成功時だけ`out`を`(high << 4) | low`へ更新。失敗時は引数と外部状態の変更なし。
Example
std::uint8_t value = 0x00U;
const auto ok = ket::bits::TryPackNibbles(0x0AU, 0x0BU, value);
// ok == true
// value == 0xAB

TryMask function

Signature
template <typename T> bool TryMask(unsigned width, T& out) noexcept
Brief下位`width` bitが1のmask生成。
Parameters
  • in width: 1にする下位bit数。
  • out out: 成功時のmask値。失敗時は変更なし。
Return / retval
  • true: `width`が`TypeBitWidth<T>()`以下で、`out`を更新。

Post成功時だけ`out`をmask値へ更新。`width == 0`は0、`width == TypeBitWidth<T>()`は全bit 1。
Example
std::uint8_t mask = 0xEEU;
const auto ok = ket::bits::TryMask<std::uint8_t>(4U, mask);
// ok == true
// mask == 0x0F
