Use This When

  • C++17以前でindex付きrange走査を短く書きたい時。
  • 条件を満たす最初の要素indexを出力引数で受けたい時。

Do Not Use This When

  • iterator pair overloadやranges frameworkが必要な時。
  • C++20 std::ranges algorithmだけで十分明確な時。

API Details

ForEachWithIndex function

Signature
template <typename Range, typename F> void ForEachWithIndex(Range&& range, F&& f)
Briefrange要素へのindex付き順次処理。
Parameters
  • in range: `begin`と`end`で走査するrange object。ADLによるfree functionも対象。
  • in,out f: 各要素へ適用するcallable。`f(index, element)`の形で同じobjectを呼び出す。
Return / retval
  • void: 戻り値なし。
Pre`range`は走査中に有効なbegin/endを提供する。走査中にiteratorを無効化する変更は 呼び出し側の責任。走査する要素数は`std::size_t`で表現可能。
Post`f`の副作用を除き、module側では`range`と外部状態を変更しない。
Notes
  • indexは0から始まり、要素順に1ずつ増加。
  • `element`は`*it`の参照性とconst性を保持。`f`の例外は捕捉せず伝播。`f`はAPI内でcopyしない。
Example
std::vector<int> values = {10, 20};
ket::ranges::ForEachWithIndex(values, [](std::size_t index, int& value) {
    value += static_cast<int>(index);
});
// values == {10, 21}

FindIndexIf function

Signature
template <typename Range, typename Predicate> bool FindIndexIf(Range&& range, Predicate&& predicate, std::size_t& out)
Brief条件を満たす最初の要素index取得。
Parameters
  • in range: `begin`と`end`で走査するrange object。ADLによるfree functionも対象。
  • in,out predicate: 各要素へ適用する述語。`predicate(element)`の形で同じobjectを呼び出す。
  • out out: 最初に条件を満たした要素の0始まりindex。
Return / retval
  • true: 条件を満たす要素が見つかり、`out`へindexを書き込み。
  • false: 条件を満たす要素なし。`out`は入力時の値を保持。
Pre`range`は走査中に有効なbegin/endを提供する。走査中にiteratorを無効化する変更は 呼び出し側の責任。走査する要素数は`std::size_t`で表現可能。
Post`predicate`の副作用を除き、module側は成功時のみ`out`を変更。not found時と例外伝播時は`out`を変更しない。
Notes
  • 最初の一致で短絡。`predicate`の例外は捕捉せず伝播。`predicate`はAPI内でcopyしない。
Example
const std::vector<int> values = {10, 20, 30};
std::size_t index = 0U;
const auto found = ket::ranges::FindIndexIf(values, [](int value) {
    return value >= 20;
}, index);
// found == true, index == 1
