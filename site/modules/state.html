ket::state

状態遷移tableのlookup、許可判定、次状態取得を扱う。

Use This When

  • 小さいFSMの遷移表をコード上で見える形にしたい時。
  • 未定義遷移を失敗として扱いたい時。

Do Not Use This When

  • 階層状態機械やevent frameworkが必要な時。
  • 大量の状態を生成コードで管理したい時。

現在状態とイベントに対応する遷移の有無判定。

Parameters

  • in current: 現在状態。
  • in event: 発生イベント。
  • in table: 検索対象の状態遷移表。

Return / retval

  • true: `table`内に一致する遷移あり。
  • false: `table`内に一致する遷移なし。

Contract

prepost
`State`と`Event`は等価比較可能。`table`は`N`行読み取り可能な配列。引数と外部状態の変更なし。比較演算が例外を送出した場合は呼び出し元へ伝播。

Notes

  • duplicate行がある場合は先頭の一致行を採用するため、判定結果はtrueで変わらない。

Example

enum class Mode { kIdle, kRunning };
enum class Event { kStart };
const ket::state::Transition<Mode, Event> table[] = {
    {Mode::kIdle, Event::kStart, Mode::kRunning},
};
const auto allowed = ket::state::IsAllowed(Mode::kIdle, Event::kStart, table);
// allowed == true

Notes

Example

enum class Mode { kIdle, kRunning };

    {Mode::kIdle, Event::kStart, Mode::kRunning},

// next.has_value() == true, *next == Mode::kRunning
