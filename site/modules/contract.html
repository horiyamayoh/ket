ket::contract

precondition、postcondition、invariantをNDEBUGに連動せず評価する。

Use cases後始末、期限、契約を局所化する 失敗時に読みやすい診断を出す

Use This When

  • debug build限定では困る前提違反を即時検出したい時。
  • nullやboundsの前提を呼び出し箇所に明示したい時。

Do Not Use This When

  • 回復可能な入力エラーを表現したい時。
  • 例外で契約違反を通知したい時。

Fail function

Signature
[[noreturn]] inline void Fail(Kind kind, const char* expression, const char* file, int line) noexcept
Brief契約違反としてprocess terminationを実行。
Parameters
  • in kind: 契約違反の種類。
  • in expression: 文字列化済みの契約式。`nullptr`でもtermination方針は不変。
  • in file: 呼び出し元file名。`nullptr`でもtermination方針は不変。
  • in line: 呼び出し元line番号。
Return / retval
  • none: 戻らない。
Preなし。診断文字列の生成や出力には依存しない。
Post`std::terminate`を呼び出し、呼び出し元へ戻らない。
Notes
  • `kind`、`expression`、`file`、`line`は将来の診断拡張用に受け取るが、 初回APIではhandler差し替えやloggingを提供しない。
Example
const auto kind = ket::contract::Kind::kExpects;
ket::contract::Fail(kind, "count <= capacity", "file.cpp", 42);
// 呼び出し元へ戻らない。

  • ptr: `ptr != nullptr`の場合、入力と同じpointer値。
  • none: `ptr == nullptr`の場合、戻らない。
Pre`ptr`はraw pointer。smart pointerやoptional pointerのunwrapは呼び出し側の責務。
Post成功時は引数と外部状態の変更なし。失敗時は`std::terminate`を呼び出す。
Example
int* const value = KET_REQUIRE_NON_NULL(ptr);
