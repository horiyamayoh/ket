ket::contract

precondition、postcondition、invariantをNDEBUGに連動せず評価する。

Use This When

  • debug build限定では困る前提違反を即時検出したい時。
  • nullやboundsの前提を呼び出し箇所に明示したい時。

Do Not Use This When

  • 回復可能な入力エラーを表現したい時。
  • 例外で契約違反を通知したい時。

Kind type

enum class Kind

契約違反の種類。

Fail function

[[noreturn]] inline void Fail(Kind kind, const char* expression, const char* file, int line) noexcept

契約違反としてprocess terminationを実行。

Parameters

  • in kind: 契約違反の種類。
  • in expression: 文字列化済みの契約式。`nullptr`でもtermination方針は不変。
  • in file: 呼び出し元file名。`nullptr`でもtermination方針は不変。
  • in line: 呼び出し元line番号。

Return / retval

  • none: 戻らない。

Contract

prepost
なし。診断文字列の生成や出力には依存しない。`std::terminate`を呼び出し、呼び出し元へ戻らない。

Notes

  • `kind`、`expression`、`file`、`line`は将来の診断拡張用に受け取るが、 初回APIではhandler差し替えやloggingを提供しない。

Example

const auto kind = ket::contract::Kind::kExpects;
ket::contract::Fail(kind, "count <= capacity", "file.cpp", 42);
// 呼び出し元へ戻らない。

inline void Expects(bool condition, const char* expression, const char* file, int line) noexcept

Parameters

Contract

prepost
なし。`condition == false`は契約違反として扱う。`condition == true`なら外部状態の変更なし。`condition == false`なら戻らない。

Example

RequireNonNull function

template <typename T> T* RequireNonNull(T* ptr, const char* expression, const char* file, int line) noexcept

non-null pointerの取得。

Parameters

  • in ptr: 検査対象pointer。
  • in expression: 文字列化済みのpointer式。`nullptr`でも失敗時はterminate。
  • in file: 呼び出し元file名。`nullptr`でも失敗時はterminate。
  • in line: 呼び出し元line番号。

Return / retval

  • ptr: `ptr != nullptr`の場合、入力と同じpointer値。
  • none: `ptr == nullptr`の場合、戻らない。

Contract

prepost
`ptr`はraw pointer。smart pointerやoptional pointerのunwrapは呼び出し側の責務。成功時は引数と外部状態の変更なし。失敗時は`std::terminate`を呼び出す。

Example

int* const value = KET_REQUIRE_NON_NULL(ptr);

IsInBounds function

constexpr bool IsInBounds(std::size_t index, std::size_t size) noexcept

indexがsize境界内かを判定。

Parameters

  • in index: 判定対象index。
  • in size: 有効範囲の要素数。

Return / retval

  • true: `index < size`。
  • false: `index >= size`。

Contract

prepost
なし。`size == 0`の場合は常にfalse。引数と外部状態の変更なし。

Notes

  • C++11 constexprとして利用可能。

Example

KET_EXPECTS(ket::contract::IsInBounds(index, size));
