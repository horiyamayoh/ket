ket::concurrency

join忘れ防止の所有threadとfuture ready判定を小さいAPIにする。

Use This When

  • std::threadのjoin忘れを局所的な所有型で避けたい時。
  • futureがreadyかをtimeoutなしで短く確認したい時。

Do Not Use This When

  • thread pool、executor、schedulerが必要な時。
  • C++20以降でstd::jthreadが用途に十分合う時。

Views

制御・状態・失敗後始末、期限、契約を局所化する

API Details

JoiningThread type

class JoiningThread

joinableなstd::threadをdestructorでjoinする所有型。

Notes

  • C++20以降で同じ用途を満たせる場合はstd::jthreadを優先。

JoiningThread function

JoiningThread() noexcept = default

threadを所有しない状態の構築。

Contract

prepost
なし。`Joinable()`はfalse。

Example

ket::concurrency::JoiningThread thread;
const auto joinable = thread.Joinable();
// joinable == false

JoiningThread function

explicit JoiningThread(std::thread thread) noexcept

std::threadの所有権取得。

Parameters

  • in thread: 所有対象のstd::thread。joinableでないthreadも受け付ける。

Contract

prepost
`thread`がjoinableな場合、destructorや代入でself-joinにならない寿命関係。`thread`の移動元状態を所有。引数のthread objectは移動元として破棄。

Example

int value = 0;
ket::concurrency::JoiningThread thread(std::thread([&value] { value = 1; }));
const auto joinable = thread.Joinable();
// joinable == true

~JoiningThread function

~JoiningThread() noexcept

所有threadのjoin。

Contract

prepost
所有threadがjoinableな場合、destructorを実行するthread自身ではない。joinableなthreadを所有していた場合は完了まで待機。

Notes

  • joinできない状態やjoin例外はnoexcept destructorによりstd::terminate。

Example

int value = 0;
{
    ket::concurrency::JoiningThread thread(std::thread([&value] { value = 1; }));
}
// value == 1

JoiningThread function

JoiningThread(const JoiningThread& other) = delete

copy構築禁止。

Parameters

  • in other: copy元。使用不可。

Contract

prepost
copy操作は利用不可。状態変更なし。

Example

const auto copyable =
std::is_copy_constructible<ket::concurrency::JoiningThread>::value;
// copyable == false

JoiningThread function

JoiningThread(JoiningThread&& other) noexcept

move構築。

Parameters

  • in,out other: 移動元のJoiningThread。

Contract

prepost
移動元は有効なJoiningThread。所有threadがjoinableな場合、移動先のdestructorでself-joinにならない寿命関係。`other`はjoinableなthreadを所有しない。

Example

ket::concurrency::JoiningThread source(std::thread([] {}));
ket::concurrency::JoiningThread thread(std::move(source));
const auto moved = thread.Joinable();
// moved == true

Joinable function

bool Joinable() const noexcept

所有threadのjoin可能状態確認。

Return / retval

  • true: 所有threadがjoinable。
  • false: 所有threadがjoinableでない。

Contract

prepost
なし。`*this`と外部状態の変更なし。

Example

ket::concurrency::JoiningThread thread;
const auto joinable = thread.Joinable();
// joinable == false

IsReady function

template <typename Future> bool IsReady(const Future& future)

future-like objectのready判定。

Parameters

  • in future: ready判定対象のfuture-like object。

Return / retval

  • true: `wait_for(0)`が`std::future_status::ready`。
  • false: `wait_for(0)`がtimeoutまたはdeferred。

Contract

prepost
`future.valid()`がtrue。invalid futureはprecondition違反。futureの共有状態を消費せず、値や例外は取り出さない。

Notes

  • `std::future_status::deferred` はreadyではないためfalse。
  • `wait_for`が例外を送出するfuture-like型では、その例外を呼び出し元へ伝播。

Example

std::promise<int> promise;
auto future = promise.get_future();
promise.set_value(42);
const auto ready = ket::concurrency::IsReady(future);
// ready == true
