ket::concurrency

join忘れ防止の所有threadとfuture ready判定を小さいAPIにする。

Use This When

  • std::threadのjoin忘れを局所的な所有型で避けたい時。
  • futureがreadyかをtimeoutなしで短く確認したい時。

Do Not Use This When

  • thread pool、executor、schedulerが必要な時。
  • C++20以降でstd::jthreadが用途に十分合う時。

JoiningThread function

Signature
explicit JoiningThread(std::thread thread) noexcept
Briefstd::threadの所有権取得。
Parameters
  • in thread: 所有対象のstd::thread。joinableでないthreadも受け付ける。
Pre`thread`がjoinableな場合、destructorや代入でself-joinにならない寿命関係。
Post`thread`の移動元状態を所有。引数のthread objectは移動元として破棄。
Example
int value = 0;
ket::concurrency::JoiningThread thread(std::thread([&value] { value = 1; }));
const auto joinable = thread.Joinable();
// joinable == true

~JoiningThread function

Signature
~JoiningThread() noexcept
Brief所有threadのjoin。
Pre所有threadがjoinableな場合、destructorを実行するthread自身ではない。
Postjoinableなthreadを所有していた場合は完了まで待機。
Notes
  • joinできない状態やjoin例外はnoexcept destructorによりstd::terminate。
Example
int value = 0;
{
    ket::concurrency::JoiningThread thread(std::thread([&value] { value = 1; }));
}
// value == 1

JoiningThread function

Signature

Briefmove構築。
Parameters
  • in,out other: 移動元のJoiningThread。
Pre移動元は有効なJoiningThread。所有threadがjoinableな場合、移動先のdestructorでself-joinにならない寿命関係。
Post`other`はjoinableなthreadを所有しない。
Example

// moved == true

IsReady function

Signature
template <typename Future> bool IsReady(const Future& future)
Brieffuture-like objectのready判定。
Parameters
  • in future: ready判定対象のfuture-like object。
Return / retval
  • true: `wait_for(0)`が`std::future_status::ready`。
  • false: `wait_for(0)`がtimeoutまたはdeferred。
Pre`future.valid()`がtrue。invalid futureはprecondition違反。
Postfutureの共有状態を消費せず、値や例外は取り出さない。
Notes
  • `std::future_status::deferred` はreadyではないためfalse。
  • `wait_for`が例外を送出するfuture-like型では、その例外を呼び出し元へ伝播。
Example
std::promise<int> promise;
auto future = promise.get_future();
promise.set_value(42);
const auto ready = ket::concurrency::IsReady(future);
// ready == true
