ket::mac

区切り付きMAC address文字列と固定6 byte値を相互変換する。

Use This When

  • MAC address literalを範囲確認して値として扱いたい時。
  • colon区切りまたはhyphen区切りのparse規則を固定したい時。

Do Not Use This When

  • vendor OUI lookupやnetwork interface列挙が必要な時。
  • IPv4/IPv6 endpoint全体を扱いたい時。

Parse function

std::optional<Address> Parse(std::string_view text) noexcept

区切り付きMAC address文字列の値変換。

Parameters

  • in text: 変換対象のMAC address文字列。

Return / retval

  • value: 変換後のMAC address値。
  • std::nullopt: byte数不足/過多、不正hex、不正区切り、混在区切り、またはCisco形式。

Contract

prepost
なし。不正な入力表記は失敗値として扱う。引数と外部状態の変更なし。

Notes

  • `:` または `-` のどちらか一方で区切られた6 byte形式のみを受け付ける。

Example

const auto value = ket::mac::Parse("AA:BB:CC:DD:EE:FF");
// value->bytes == {0xAA, 0xBB, 0xCC, 0xDD, 0xEE, 0xFF}

Format function

std::string Format(Address value, LetterCase letter_case = LetterCase::kLower)

MAC address値のcolon区切り文字列変換。

Parameters

  • in value: 変換対象のMAC address値。
  • in letter_case: hex digitの大文字小文字。

Return / retval

  • value: colon区切りのMAC address文字列。

Contract

prepost
なし。引数と外部状態の変更なし。出力は常に6 byte、colon区切り、2桁hexの正準形。

Notes

  • `letter_case` が `LetterCase::kUpper` の場合は大文字、それ以外は小文字で出力。
  • std::stringの確保があるためnoexceptなし。

Example

ket::mac::Address address{{0xAA, 0xBB, 0xCC, 0xDD, 0xEE, 0xFF}};
const auto text = ket::mac::Format(address);
// text == "aa:bb:cc:dd:ee:ff"
