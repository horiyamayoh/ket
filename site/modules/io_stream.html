ket::io_stream

short read/write判定、行末trim付き行読み、書式状態のscope復元を扱う。

Use This When

  • streamから指定バイト数を読み切る、または書き切る意図を明示したい時。
  • iostreamの書式状態をscope終了時に戻したい時。

Do Not Use This When

  • file全体を一括で読み書きしたい時。
  • 非同期I/Oや独自buffering layerが必要な時。

API Details

TryReadExactly function

bool TryReadExactly(std::istream& stream, std::uint8_t* data, std::size_t size)

指定バイト数のstream読み切り。

Parameters

  • in,out stream: 読み取り元stream。
  • out data: 読み取り先バッファ。
  • in size: 読み取るバイト数。

Return / retval

  • true: `size`バイトを読み切った。
  • false: `size > 0 && data == nullptr`、short read、stream errorのいずれかで`size`バイトを読み切れなかった。

Contract

prepost
`size == 0`では`data == nullptr`を許可。`size > 0`で`data != nullptr`の場合、 `data`は`size`バイト以上書き込み可能な領域を指す。成功時は`data[0, size)`へ読み取ったバイト列を格納。失敗時はstream状態と `data`の一部が標準istreamのread結果に従って変化する場合あり。`data == nullptr && size > 0`の場合はstreamへアクセスせず`false`。

Notes

  • streamのexception maskにより、読み取り中の例外は呼び出し側へ伝播。

Example

std::uint8_t bytes[4] = {};
const auto ok = ket::io_stream::TryReadExactly(stream, bytes, 4U);
// ok == true のとき bytes[0..3] が読み取り済み

TryWriteAll function

bool TryWriteAll(std::ostream& stream, const std::uint8_t* data, std::size_t size)

指定バイト数のstream書き切り。

Parameters

  • in,out stream: 書き込み先stream。
  • in data: 書き込み元バッファ。
  • in size: 書き込むバイト数。

Return / retval

  • true: `size`バイトを書き切った。
  • false: `size > 0 && data == nullptr`、またはstream errorにより`size`バイトを書き切れなかった。

Contract

prepost
`size == 0`では`data == nullptr`を許可。`size > 0`で`data != nullptr`の場合、 `data`は`size`バイト以上読み取り可能な領域を指す。成功時は`data[0, size)`のバイト列をstreamへ書き込み済み。失敗時はstream状態と 書き込み済み範囲が標準ostreamのwrite結果に従う。`data == nullptr && size > 0`の場合は streamへアクセスせず`false`。

Notes

  • streamのexception maskにより、書き込み中の例外は呼び出し側へ伝播。

Example

const std::uint8_t bytes[] = {0x01U, 0x02U};
const auto ok = ket::io_stream::TryWriteAll(stream, bytes, 2U);
// ok == true のとき2バイトを書き込み済み

TryReadLineTrimRightAscii function

bool TryReadLineTrimRightAscii(std::istream& stream, std::string& out)

行末ASCII whitespace除去付きの1行読み取り。

Parameters

  • in,out stream: 読み取り元stream。
  • out out: 読み取った行の格納先。

Return / retval

  • true: 1行を読み取った。空行と改行なしでEOFに到達した最終行も成功として扱う。
  • false: EOFで1文字も読めない、またはstream error。

Contract

prepost
`out`は有効なstd::string object。成功時だけ`out`を更新し、末尾のASCII whitespaceを除去した行を格納。失敗時は `out`を変更なし。先頭のASCII whitespaceは保持。

Notes

  • 除去対象はASCIIのspace、tab、CR、LF、form feed、vertical tabのみ。std::getlineが delimiterとして消費したLFは行文字列に含まれない。
  • std::stringのallocationとstream exceptionの可能性があるためnoexceptなし。

Example

std::string line;
const auto ok = ket::io_stream::TryReadLineTrimRightAscii(stream, line);
// 入力 "value \t\r\n" では ok == true, line == "value"

FormatStateSaver type

class FormatStateSaver

streamの書式状態をscope終了時に復元するRAII object。

Notes

  • 保存対象はflags、precision、fill。error state、exception mask、locale、tie、 widthは保存対象外。copy/moveは禁止。

FormatStateSaver function

explicit FormatStateSaver(std::ios& stream)

stream書式状態の保存開始。

Parameters

  • in,out stream: 保存と復元の対象stream。

Contract

prepost
`stream`はこのobjectのdestructor完了まで同じobjectとして生存。`stream`のflags、precision、fillを記録。構築直後のstream状態は変更なし。

Example

std::ostringstream stream;
const ket::io_stream::FormatStateSaver saver(stream);
stream.precision(9);

~FormatStateSaver function

~FormatStateSaver() noexcept

保存したstream書式状態の復元。

Contract

prepost
対象streamはconstructor呼び出し時と同じobjectとして生存。対象streamのflags、precision、fillはconstructor呼び出し時の値へ復元。

Example

std::ostringstream stream;
stream.precision(3);
{
    const ket::io_stream::FormatStateSaver saver(stream);
    stream.precision(9);
}
// stream.precision() == 3
