ket::file

通常ファイルの全体読み込み、全体書き込み、存在確認、サイズ取得を扱う。

Namespace公開API：ket::file 内部実装：.cpp の無名 namespace

Use This When

  • 小さい通常ファイルをtextまたはbytesとして丸ごと扱いたい時。
  • path存在、directory判定、file size取得を標準型で短く書きたい時。

Do Not Use This When

  • streaming、atomic write、再帰処理が必要な時。
  • path正規化、encoding変換、watcherを扱いたい時。

Views

入力境界・外部境界固定bufferから安全に読む・書く CLI引数や設定文字列を読む 文字列の小処理を読みやすくする

API Details

TryReadAllText function

bool TryReadAllText(const std::filesystem::path& path, std::string& out, std::error_code* error = nullptr)

ファイル全体のtext読み込み。

Parameters

  • in path: 読み込み対象path。
  • out out: 読み込んだbytesを格納する文字列。
  • out error: 失敗詳細のoptional出力。`nullptr`の場合は詳細を破棄。

Return / retval

  • true: ファイル全体を読み込み、`out`を更新。
  • false: path不在、通常ファイル以外、permission、短いread、読み込み中の内容増加、またはサイズ過大。

Contract

prepost
`out`は有効なstd::string。`error`は`nullptr`または有効なstd::error_code。成功時だけ`out`を置換。失敗時は`out`不変。成功時は`error`をclearし、失敗時は可能な範囲で原因を設定。

Notes

  • text encoding変換なし。ファイルbytesをそのままstd::stringへ保持。
  • 読み込み中の同時更新に対するsnapshot保証なし。open後の終端を基準にし、余剰byteを検出した場合は失敗。
  • stream由来の失敗は、標準ライブラリから詳細を取得できない場合にstd::errc::io_errorで報告。
  • std::stringのallocation例外は呼び出し側へ伝播。

Example

std::string text;
std::error_code error;
const auto ok = ket::file::TryReadAllText("config.txt", text, &error);

TryWriteAllText function

bool TryWriteAllText(const std::filesystem::path& path, std::string_view text, std::error_code* error = nullptr)

text内容のファイル全体書き込み。

Parameters

  • in path: 書き込み対象path。
  • in text: 書き込むbytesを指す文字列view。
  • out error: 失敗詳細のoptional出力。`nullptr`の場合は詳細を破棄。

Return / retval

  • true: `text`全体を書き込み、stream closeまで成功。
  • false: pathを開けない、permission、既存pathが通常ファイル以外、短いwrite、またはclose失敗。

Contract

prepost
`text`は`text.size()`バイト以上読み取り可能な範囲を指す。`error`は`nullptr`または有効なstd::error_code。成功時は対象pathの内容を`text`で置換。未存在pathは新規作成。失敗時の部分書き込み有無はplatformと stream状態に従う。fileを開いた後の失敗では、既存内容がtruncate済みまたは途中まで置換済みの場合がある。

Notes

  • text encoding変換なし。`text`のbytesをそのまま書き込む。
  • stream由来の失敗は、標準ライブラリから詳細を取得できない場合にstd::errc::io_errorで報告。

Example

std::error_code error;
const auto ok = ket::file::TryWriteAllText("config.txt", "mode=run", &error);

TryWriteAllBytes function

bool TryWriteAllBytes(const std::filesystem::path& path, const std::uint8_t* data, std::size_t size, std::error_code* error = nullptr)

bytes内容のファイル全体書き込み。

Parameters

  • in path: 書き込み対象path。
  • in data: 書き込むbytesの先頭。`size == 0`の場合だけ`nullptr`可。
  • in size: 書き込むbyte数。
  • out error: 失敗詳細のoptional出力。`nullptr`の場合は詳細を破棄。

Return / retval

  • true: `data[0, size)`全体を書き込み、stream closeまで成功。
  • false: pathを開けない、permission、既存pathが通常ファイル以外、短いwrite、close失敗、またはnonnull条件違反。

Contract

prepost
sizeが0より大きい場合、`data`は`size`バイト以上読み取り可能な配列を指す。`error`は`nullptr`または有効なstd::error_code。成功時は対象pathの内容を入力bytesで置換。未存在pathは新規作成。失敗時の部分書き込み有無はplatformと stream状態に従う。fileを開いた後の失敗では、既存内容がtruncate済みまたは途中まで置換済みの場合がある。

Notes

  • raw pointerはbytes列のC API境界として扱うため採用。
  • stream由来の失敗は、標準ライブラリから詳細を取得できない場合にstd::errc::io_errorで報告。

Example

const std::uint8_t payload[] = {0x01U, 0x02U};
const auto ok = ket::file::TryWriteAllBytes("payload.bin", payload, 2U);
