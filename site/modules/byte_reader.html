ket::byte_reader

non-owning bufferから成功時だけoffsetを進めてbyte列を読む。

Use cases固定bufferから安全に読む・書く 小さい電文payloadを組み立てる

Use This When

  • 固定bufferから順にfieldを読みたい時。
  • size不足時にoffsetと出力を不変にしたい時。

Do Not Use This When

  • stream I/Oや非同期I/Oを扱う時。
  • buffer所有や再確保が必要な時。

Reader function

Signature
Reader(const std::uint8_t* data, std::size_t size) noexcept
Briefnon-owning buffer readerの構築。
Parameters
  • in data: 読み取り対象buffer先頭。`size == 0`の場合のみnullptr可。
  • in size: `data`のbyte数。
Pre`data != nullptr`の場合、`data`は`size`byte以上読み取り可能な配列を指す。 `data == nullptr && size > 0`はprecondition違反ではなくinvalid readerとして保持。
Post`Offset()`は0。入力pointerとsizeを保持。
Example
const std::uint8_t data[] = {0x12U, 0x34U};
ket::byte_reader::Reader reader(data, 2U);
// reader.Size() == 2
// reader.Offset() == 0

ReadBytes function

Signature
bool ReadBytes(std::size_t size, const std::uint8_t*& out_data) noexcept
Brief指定byte数のnon-owning範囲読み取り。
Parameters
  • in size: 取り出すbyte数。
  • out out_data: 読み取り開始位置を指すnon-owning pointer。
Return / retval
  • true: `size`byteの範囲読み取り成功。
  • false: invalid reader、または残りbyte不足。
Pre`out_data`は有効な参照。返されたpointerのlifetimeは元bufferに従う。
Post成功時だけ`Offset()`が`size`増加し、`out_data`へ元buffer内の位置を書き込む。 失敗時はoffsetと`out_data`不変。
Notes
  • validな空readerから0byteを読む場合、`out_data`はnullptr。
Example
const std::uint8_t data[] = {0x10U, 0x20U, 0x30U};
ket::byte_reader::Reader reader(data, 3U);
const std::uint8_t* bytes = nullptr;
const auto ok = reader.ReadBytes(2U, bytes);
// ok == true, bytes == data, reader.Offset() == 2
