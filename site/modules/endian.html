ket::endian

BE/LEと幅をAPI名に出した16/32/64bit読み書き。

Module Facts

Headermodules/endian/ket_endian.h
Statusverified
C++ MinC++11
TestsGoogleTest + C++11 compile-only
Progress notebyte orderを明示した16/32/64bit読み書き
Drop-in`ket_endian.h` と `ket_endian.cpp` を対象プロジェクトへコピー。
Dependencies標準ライブラリのみ。 他のket moduleへの依存なし。
Namespace公開API：ket::endian 内部実装：ket::endian::detail

Use This When

  • unaligned bufferから固定幅整数を読み書きしたい時。
  • byte orderを呼び出し側の名前で明示したい時。

Do Not Use This When

  • stream cursorやoffset管理も必要な時。
  • host endian変換の大規模互換layerが必要な時。

Views

バイナリ・byte列固定bufferから安全に読む・書く 小さい電文payloadを組み立てる

Related Modules

byte_reader byte_writer bytes bits

Notes

  • 固定位置の変換に絞り、cursor管理はreader/writerへ分ける。

LoadBe16 function

std::uint16_t LoadBe16(const std::uint8_t* data) noexcept

big-endian 2byte列の16bit整数読み取り。

Parameters

  • in data: 読み取り対象の先頭byte。

Return / retval

  • value: `data[0]`を上位byte、`data[1]`を下位byteとして組み立てた値。

Contract

prepost
`data`は2byte以上読み取り可能な領域を指す。nullptrはprecondition違反。引数と外部状態の変更なし。

Notes

  • 実装はbyte単位の読み取りとshift/orのみを使い、reinterpret_castとunaligned accessに依存しない。

Example

const std::uint8_t data[] = {0x12U, 0x34U};
const auto value = ket::endian::LoadBe16(data);
// value == 0x1234

StoreBe16 function

void StoreBe16(std::uint8_t* data, std::uint16_t value) noexcept

16bit整数のbig-endian 2byte列書き込み。

Parameters

  • in,out data: 書き込み対象の先頭byte。
  • in value: 書き込む値。

Return / retval

  • void: 戻り値なし。

Contract

prepost
`data`は2byte以上書き込み可能な領域を指す。nullptrはprecondition違反。`data[0]`と`data[1]`に`value`のbig-endian表現を書き込む。

Notes

  • 実装はbyte単位の書き込みとshift/maskのみを使い、reinterpret_castとunaligned accessに依存しない。

Example

std::uint8_t data[2] = {};
ket::endian::StoreBe16(data, static_cast<std::uint16_t>(0x1234U));
// data[0] == 0x12 && data[1] == 0x34

TryLoadBe16 function

bool TryLoadBe16(const std::uint8_t* data, std::size_t size, std::uint16_t& out) noexcept

big-endian 2byte列の16bit整数読み取り試行。

Parameters

  • in data: 読み取り対象の先頭byte。nullptrは失敗値として扱う。
  • in size: `data`から読み取り可能なbyte数。
  • out out: 読み取り成功時の出力先。

Return / retval

  • true: 読み取り成功。
  • false: `data`がnullptr、または`size < 2`。

Contract

prepost
`data`がnullptrでない場合、`data`は`size`byte以上読み取り可能な領域を指す。成功時だけ`out`を変更。失敗時は`out`と外部状態の変更なし。

Example

const std::uint8_t data[] = {0x12U, 0x34U};
std::uint16_t value = 0U;
const auto ok = ket::endian::TryLoadBe16(data, sizeof(data), value);
// ok == true && value == 0x1234

  • in,out data: 書き込み対象の先頭byte。nullptrは失敗値として扱う。
  • in size: `data`へ書き込み可能なbyte数。
  • in value: 書き込む値。

Return / retval

  • true: 書き込み成功。
  • false: `data`がnullptr、または`size < 8`。

Contract

prepost
`data`がnullptrでない場合、`data`は`size`byte以上書き込み可能な領域を指す。成功時だけbufferを変更。失敗時はbufferと外部状態の変更なし。

Example

std::uint8_t data[8] = {};
const auto ok = ket::endian::TryStoreBe64(data, sizeof(data), 0x0123456789ABCDEFULL);
// ok == true && data[0] == 0x01 && data[7] == 0xEF
