map/vectorの小さいlookup、生成、削除補助を集める。

Use This When

  • findやerase-removeの儀式が呼び出し側の意図を隠す時。
  • 小さいcontainer操作を名前で読みたい時。

Do Not Use This When

  • 標準algorithmをそのまま使う方が明確な時。
  • 複雑なcollection abstractionを作りたい時。

Contains function

template <typename Container, typename Value> bool Contains(const Container& container, const Value& value)

range内の値存在確認。

Parameters

  • in container: 検索対象のコンテナ。
  • in value: 検索値。

Return / retval

  • true: `value`と等値比較される要素あり。
  • false: `value`と等値比較される要素なし。

Contract

prepost
`container`は有効なrange。要素と`value`の等値比較が有効。引数と外部状態の変更なし。iterator取得や等値比較の例外は呼び出し元へ伝播。

Example

const auto found = ket::container::Contains(values, 42);
// found == true if values contains 42

AtOr function

template <typename Map, typename Key> typename Map::mapped_type AtOr(const Map& map, const Key& key, typename Map::mapped_type default_value)

map内の値またはdefault値取得。

Parameters

  • in map: 検索対象のmap。
  • in key: 検索key。
  • in default_value: `key`が無い場合に返す値。

Return / retval

  • value: `key`があればmap内要素のcopy、無ければ`default_value`。

prepost
`map.find(key)`が有効。`Map::mapped_type`はcopy constructible。引数と外部状態の変更なし。lookupやcopy/moveの例外は呼び出し元へ伝播。

Example

const std::map<std::string, int> values = {{"one", 1}};
const auto value = ket::container::AtOr(values, std::string("missing"), 7);
// value == 7

AtOrCreate function

template <typename Map, typename Key, typename Factory> typename Map::mapped_type& AtOrCreate(Map& map, const Key& key, Factory factory)

map内の値取得またはfactory生成。

Parameters

  • in,out map: 検索と必要時の挿入対象。
  • in key: 検索key。
  • in factory: `key`が無い場合だけ呼ぶ生成処理。

Return / retval

  • value: 既存または新規挿入されたmap内要素への参照。

Contract

prepost
`map.find(key)`、`Map::key_type`の`key`からの構築、factory結果の`Map::mapped_type` 化、`map.emplace(...)`が有効。factoryは`map`を変更しない。既存keyではmap構造を変更せず、factory呼び出しなし。keyが無い場合はkeyをmaterialize してからfactoryを呼び、生成値を挿入。生成、lookup、挿入の例外は呼び出し元へ伝播。

Example

std::map<std::string, int> values;
int& value = ket::container::AtOrCreate(values, std::string("one"), [] { return 1; });
// value == 1 && values.size() == 1

EraseIf function

template <typename Sequence, typename Predicate> std::size_t EraseIf(Sequence& sequence, Predicate predicate)

条件に一致するsequence要素の削除。

Parameters

  • in,out sequence: 削除対象のsequence。
  • in predicate: 削除対象ならtrueを返す述語。

Return / retval

  • value: 削除した要素数。

Contract

prepost
`std::remove_if`と`sequence.erase(first, last)`が有効。associative containerは対象外。成功時は条件一致要素を削除し、相対順序は標準erase-removeの性質に従う。 述語や要素移動の例外は呼び出し元へ伝播し、例外時のsequence状態は標準アルゴリズムの規則に従う。

Example

std::vector<int> values = {1, 2, 3, 4};
const auto removed = ket::container::EraseIf(values, [](int value) {
    return value % 2 == 0;
});
// removed == 2 && values == {1, 3}

SortUnique function

template <typename Vector> void SortUnique(Vector& values)

vector-like値列のsortと重複削除。

Parameters

  • in,out values: sortと重複削除の対象。

Return / retval

  • void: 戻り値なし。

Contract

prepost
`std::sort`、`std::unique`、`values.erase(first, last)`が有効。`operator<` の同値関係と`operator==`が同じ重複集合を表す。`values`は昇順に並び、等値の重複要素は1つだけ残る。 比較、移動、eraseの例外は呼び出し元へ伝播。

Example

std::vector<int> values = {3, 1, 3, 2};
ket::container::SortUnique(values);
// values == {1, 2, 3}
