ket::byte_view

所有しないbyte範囲とslice境界を明示する。

Use This When

  • byte配列を所有せずに渡したい時。
  • nullptr+0とnullptr+非0の意味を固定したい時。

Do Not Use This When

  • payloadを所有して構築したい時。
  • C++20 std::span が利用でき、制約が十分に表現できる時。

size_ function

Signature
constexpr View(const std::uint8_t* data, std::size_t size) noexcept : data_(data), size_(size)
Brief読み取り専用byte列を参照するview構築。
Parameters
  • in data: 参照対象bufferの先頭。`nullptr`は`size == 0`の場合のみ空view。
  • in size: 参照対象bufferのbyte数。
Pre`data`は`size`byte以上読み取り可能な配列を指す。`nullptr + 非0`はinvalid viewとして保持。
Post`data`と`size`を保持し、bufferの所有権は取得なし。
Example
const std::uint8_t data[] = {0x10U, 0x20U};
ket::byte_view::View view(data, 2U);
// view.Data() == data
// view.Size() == 2

TryAt function

Signature
bool TryAt(std::size_t index, std::uint8_t& out) const noexcept
Brief指定indexのbyte取得。
Parameters
  • in index: 取得対象の0始まりindex。
  • out out: 取得したbyteの格納先。
Return / retval

ket::byte_view::View view(data, 2U);
std::uint8_t value = 0U;
const bool ok = view.TryAt(1U, value);
// ok == true, value == 0x20
