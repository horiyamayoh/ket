ket::memory

alignment判定、secure zero、object representationの小さい補助。

Use This When

  • pointer alignmentやobject bytesの意図を名前で明示したい時。
  • secure zeroをbest-effortで行いたい時。

Do Not Use This When

  • allocator、object lifetime、provenanceを抽象化したい時。
  • 暗号library並みの消去保証が必要な時。

IsAligned function

inline bool IsAligned(const void* ptr, std::size_t alignment) noexcept

pointerが指定alignment境界上にあるかの判定。

Parameters

  • in ptr: 判定対象のpointer。
  • in alignment: 要求alignment。power-of-twoだけを有効値として扱う。

Return / retval

  • true: `ptr`が非nullで、`alignment`がpower-of-twoかつ境界上。
  • false: `nullptr`、`alignment == 0`、非power-of-two、または境界外。

Contract

prepost
なし。null pointerと不正alignmentはfalseとして扱う。引数と外部状態の変更なし。

Notes

  • pointer値は`std::uintptr_t`へ変換して判定する。

Example

alignas(16) unsigned char buffer[16] = {};
const auto aligned = ket::memory::IsAligned(buffer, 16U);
// aligned == true

TryAlignUp function

inline bool TryAlignUp(const void* ptr, std::size_t alignment, const void*& out) noexcept

pointerを指定alignment境界へ切り上げる。

Parameters

  • in ptr: 計算対象のpointer。
  • in alignment: 要求alignment。power-of-twoだけを有効値として扱う。
  • out out: 成功時のaligned pointer。失敗時は変更なし。

Return / retval

  • true: `ptr`が非nullで、`alignment`がpower-of-twoかつ切り上げ結果を表現可能。
  • false: `nullptr`、`alignment == 0`、非power-of-two、またはaddress計算overflow。

Contract

prepost
`out`は有効な参照。null pointerと不正alignmentは失敗値として扱う。成功時だけ`out`を更新。失敗時は`out`と外部状態の変更なし。

Notes

  • pointer値は`std::uintptr_t`へ変換し、結果を`const void*`へ戻す。 返されたpointerのobject bounds、lifetime、provenance、dereference可能性は保証しない。

Example

alignas(8) unsigned char buffer[16] = {};
const void* out = nullptr;
const auto ok = ket::memory::TryAlignUp(buffer + 1U, 8U, out);
// ok == true, out == buffer + 8

Zero function

inline void Zero(void* ptr, std::size_t size) noexcept

指定byte範囲の通常zeroing。

Parameters

  • in,out ptr: zeroing対象の先頭pointer。
  • in size: zeroing対象のbyte数。

Return / retval

  • void: 戻り値なし。

Contract

prepost
`size > 0`の場合、`ptr`は`size`バイト以上書き込み可能な領域を指す。 `ptr == nullptr && size > 0`はprecondition違反。`ptr != nullptr && size > 0`の場合、対象範囲のbyteを0へ変更。 `ptr == nullptr && size == 0`はno-op。

Notes

  • `std::memset`による通常のzeroing。最適化除去への対策は`SecureZero`を使用。 `ptr == nullptr && size > 0`は呼び出し側の契約違反で、`std::terminate`を呼ぶ。

Example

unsigned char bytes[4] = {1U, 2U, 3U, 4U};
ket::memory::Zero(bytes, 4U);
// bytes == {0, 0, 0, 0}

SecureZero function

inline void SecureZero(void* ptr, std::size_t size) noexcept

指定byte範囲のbest-effort secure zeroing。

Parameters

  • in,out ptr: zeroing対象の先頭pointer。
  • in size: zeroing対象のbyte数。

Return / retval

  • void: 戻り値なし。

Contract

prepost
`size > 0`の場合、`ptr`は`size`バイト以上書き込み可能な領域を指す。 `ptr == nullptr && size > 0`はprecondition違反。`ptr != nullptr && size > 0`の場合、対象範囲のbyteを0へ変更。 `ptr == nullptr && size == 0`はno-op。

Notes

  • `volatile unsigned char*`経由のwriteで最適化除去を防ぐbest-effort。 `ptr == nullptr && size > 0`は呼び出し側の契約違反で、`std::terminate`を呼ぶ。 暗号学的な完全消去保証なし。

Example

unsigned char secret[4] = {1U, 2U, 3U, 4U};
ket::memory::SecureZero(secret, 4U);
// secret == {0, 0, 0, 0}

ObjectBytes function

template <typename T> const unsigned char* ObjectBytes(const T& object) noexcept

trivially copyable objectのobject representation先頭pointer取得。

Parameters

  • in object: 読み取り対象object。

Return / retval

  • value: `object`のobject representation先頭を指す`const unsigned char*`。

Contract

prepost
`T`はtrivially copyable。`object`のlifetime中だけ戻り値を利用可能。引数と外部状態の変更なし。object representationを読む入口だけを提供。

Notes

  • object lifetime操作、placement new、type punningは行わない。 padding、endianness、layoutを含む処理系依存のbyte列であり、serializationや安定比較には使わない。

Example

const auto value = std::uint32_t{0x01020304U};
const auto* bytes = ket::memory::ObjectBytes(value);
const auto size = ket::memory::ObjectByteSize(value);

template <typename T> constexpr std::size_t ObjectByteSize(const T& object) noexcept

trivially copyable objectのobject representation byte数取得。

Contract

prepost

Notes

  • 引数は型推論のために受け取り、値の読み取りなし。 byte列の意味はpadding、endianness、layoutに依存する。

Example
