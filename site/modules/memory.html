ket::memory

alignment判定、secure zero、object representationの小さい補助。

Use This When

  • pointer alignmentやobject bytesの意図を名前で明示したい時。
  • secure zeroをbest-effortで行いたい時。

Do Not Use This When

  • allocator、object lifetime、provenanceを抽象化したい時。
  • 暗号library並みの消去保証が必要な時。

IsAligned function

Signature
inline bool IsAligned(const void* ptr, std::size_t alignment) noexcept
Briefpointerが指定alignment境界上にあるかの判定。
Parameters
  • in ptr: 判定対象のpointer。
  • in alignment: 要求alignment。power-of-twoだけを有効値として扱う。
Return / retval
  • true: `ptr`が非nullで、`alignment`がpower-of-twoかつ境界上。
  • false: `nullptr`、`alignment == 0`、非power-of-two、または境界外。
Preなし。null pointerと不正alignmentはfalseとして扱う。
Post引数と外部状態の変更なし。
Notes
  • pointer値は`std::uintptr_t`へ変換して判定する。
Example

// aligned == true

Example

const void* out = nullptr;
const auto ok = ket::memory::TryAlignUp(buffer + 1U, 8U, out);
// ok == true, out == buffer + 8

Zero function

Signature
inline void Zero(void* ptr, std::size_t size) noexcept
Brief指定byte範囲の通常zeroing。
Parameters
  • in,out ptr: zeroing対象の先頭pointer。
  • in size: zeroing対象のbyte数。
Return / retval
  • void: 戻り値なし。
Pre`size > 0`の場合、`ptr`は`size`バイト以上書き込み可能な領域を指す。 `ptr == nullptr && size > 0`はprecondition違反。
Post`ptr != nullptr && size > 0`の場合、対象範囲のbyteを0へ変更。 `ptr == nullptr && size == 0`はno-op。
Notes
  • `std::memset`による通常のzeroing。最適化除去への対策は`SecureZero`を使用。 `ptr == nullptr && size > 0`は呼び出し側の契約違反で、`std::terminate`を呼ぶ。
Example
unsigned char bytes[4] = {1U, 2U, 3U, 4U};
ket::memory::Zero(bytes, 4U);
// bytes == {0, 0, 0, 0}

SecureZero function

Signature
inline void SecureZero(void* ptr, std::size_t size) noexcept
Brief指定byte範囲のbest-effort secure zeroing。
Parameters
  • in,out ptr: zeroing対象の先頭pointer。
  • in size: zeroing対象のbyte数。
Return / retval
  • void: 戻り値なし。
Pre`size > 0`の場合、`ptr`は`size`バイト以上書き込み可能な領域を指す。 `ptr == nullptr && size > 0`はprecondition違反。
Post`ptr != nullptr && size > 0`の場合、対象範囲のbyteを0へ変更。 `ptr == nullptr && size == 0`はno-op。
Notes
  • `volatile unsigned char*`経由のwriteで最適化除去を防ぐbest-effort。 `ptr == nullptr && size > 0`は呼び出し側の契約違反で、`std::terminate`を呼ぶ。 暗号学的な完全消去保証なし。
Example
unsigned char secret[4] = {1U, 2U, 3U, 4U};
ket::memory::SecureZero(secret, 4U);
// secret == {0, 0, 0, 0}

ObjectBytes function

Signature
template <typename T> const unsigned char* ObjectBytes(const T& object) noexcept
Brieftrivially copyable objectのobject representation先頭pointer取得。
Parameters
  • in object: 読み取り対象object。
Return / retval
  • value: `object`のobject representation先頭を指す`const unsigned char*`。
Pre`T`はtrivially copyable。`object`のlifetime中だけ戻り値を利用可能。
Post引数と外部状態の変更なし。object representationを読む入口だけを提供。
Notes
  • object lifetime操作、placement new、type punningは行わない。 padding、endianness、layoutを含む処理系依存のbyte列であり、serializationや安定比較には使わない。
Example
const auto value = std::uint32_t{0x01020304U};
const auto* bytes = ket::memory::ObjectBytes(value);
const auto size = ket::memory::ObjectByteSize(value);
