port番号の値型、範囲確認、parse/formatを扱う。

Use This When

  • 0から65535のport値を裸intから分けたい時。
  • CLIや設定値からportを読む時。

Do Not Use This When

  • service name解決やsocket addressを扱う時。
  • IPv6 endpoint全体を表したい時。

Notes

TryFromUInt function

Signature
constexpr bool TryFromUInt(std::uintmax_t value, Port& out) noexcept
Briefwide unsigned整数からport値型への範囲確認付き変換。
Parameters
  • in value: 変換対象の整数値。
  • out out: 成功時の格納先。失敗時は変更なし。
Return / retval
  • true: `value`が0から65535の範囲内で、`out`に変換結果を格納。
  • false: `value`が65535を超過し、`out`は入力時の値を保持。
Pre`out`は有効な参照。
Post成功時は`out.value == value`。失敗時は`out`を変更しない。外部状態の変更なし。
Example
ket::port::Port port;
const auto ok = ket::port::TryFromUInt(443U, port);
// ok == true, port.value == 443

Parse function

Signature
std::optional<Port> Parse(std::string_view text) noexcept
Brief10進文字列からport値型へのparse。
Parameters
  • in text: 変換対象の10進文字列。
Return / retval
  • value: 変換後のport値。
  • std::nullopt: 空文字列、空白、符号、不正文字、範囲外、または複数桁のleading zero。
Pre`text`が参照する範囲は呼び出し中読み取り可能。
Post引数と外部状態の変更なし。
Notes
  • 入力はtrimしない。10進数字だけを完全消費し、`"0"`以外のleading zeroは失敗値。
Example
const auto port = ket::port::Parse("8080");
// port.has_value() == true, port->value == 8080

Post引数と外部状態の変更なし。
Notes

Example
